# 🚀 پلتفرم تست API (API Testing Platform)

## ۱. معرفی پروژه
این پروژه یک پلتفرم جامع و قدرتمند دسکتاپ برای تست و توسعه API است که با استفاده از **Electron.js** ساخته شده است. این پلتفرم با الهام از ابزارهایی مانند Postman طراحی شده، اما با تمرکز بر استفاده محلی (Local)، آفلاین، و درون‌سازمانی، یک محیط امن و پرسرعت برای تیم‌های فنی فراهم می‌کند. 

هدف اصلی این نرم‌افزار، ارائه یک تجربه کاربری (UX) حرفه‌ای و مدرن برای توسعه‌دهندگان است تا بتوانند درخواست‌های HTTP، اتصالات WebSocket، GraphQL، و SSE را به راحتی مدیریت و تست کنند.

---

## ۲. چرایی ایجاد پروژه
دلایل اصلی توسعه این پلتفرم عبارتند از:
- **نیاز به ابزار داخلی و امن:** نگهداری داده‌ها، توکن‌ها و اطلاعات حساس APIها به صورت آفلاین و درون‌سازمانی.
- **استفاده آفلاین:** امکان تست و توسعه بدون نیاز به اتصال به اینترنت یا سرویس‌های ابری خارجی.
- **کنترل کامل روی محیط اجرای تست‌ها:** عدم وابستگی به محدودیت‌ها یا تغییرات سیاست‌های ابزارهای شخص ثالث.
- **توسعه‌پذیری بالا (Extensibility):** امکان اضافه کردن قابلیت‌های سفارشی متناسب با نیازهای خاص تیم توسعه.
- **حذف هزینه‌های اشتراک:** جایگزینی قدرتمند برای ابزارهای تجاری با قابلیت‌های مشابه.

---

## ۳. قابلیت‌های پروژه

### 🌐 مدیریت درخواست‌ها
- پشتیبانی کامل از متدهای HTTP (GET, POST, PUT, DELETE, ...)
- مدیریت Header ها، Query Parameter ها و Body (JSON, Form-Data, URLEncoded, Binary)
- پشتیبانی از پروتکل‌های مدرن: GraphQL, WebSocket, Server-Sent Events (SSE)

### 📁 کالکشن‌ها (Collections)
- سازماندهی درخواست‌ها در قالب پوشه‌ها و کالکشن‌های تودرتو.
- قابلیت Import و Export (پشتیبانی از فرمت‌های Postman و OpenAPI).

### 🌍 محیط‌ها (Environment)
- پشتیبانی از متغیرهای محلی، گلوبال و سطح محیط (Environment Variables).
- قابلیت جایگذاری متغیرها به صورت پویا (مانند `{{base-url}}`).
- پیش‌نمایش لحظه‌ای (Hover Preview) متغیرها در محیط کاربری.

### 🏃‍♂️ اجراکننده کالکشن (Runner)
- اجرای متوالی درخواست‌های یک کالکشن با قابلیت تنظیم تاخیر (Delay) و تعداد تکرار (Iterations).
- دریافت گزارش‌های دقیق از موفقیت یا شکست تست‌ها به صورت درلحظه.

### 📜 موتور اسکریپت (Script Engine)
- پشتیبانی از **Pre-request Scripts** برای تنظیم داده‌ها قبل از ارسال درخواست.
- پشتیبانی از **Test Scripts** برای اعتبارسنجی پاسخ‌ها (Response Validation).
- استفاده از ساختار مشابه Postman (مانند `pm.environment.set` و `pm.test`).

### 🧪 تست خودکار
- امکان نوشتن Assertions روی Status Code، زمان پاسخ‌گویی و محتوای Body.
- یکپارچگی با Collection Runner برای تست‌های رگرسیون (Regression Tests).

### 🌍 مدیریت زبان و فونت
- پشتیبانی کامل از چندزبانگی (انگلیسی و فارسی).
- استفاده از فونت‌های استاندارد فارسی (مانند ایران‌سنس) برای تجربه کاربری بهتر در حالت فارسی.

### 🔍 جستجو
- سیستم جستجوی قدرتمند در تاریخچه (History) و کالکشن‌ها.

### 🎨 تجربه کاربری (UX)
- رابط کاربری مدرن با تم تاریک/روشن (Dark/Light Mode).
- پنل‌های با قابلیت تغییر اندازه (Resizable Panels) مشابه IDEهای حرفه‌ای (VSCode).
- عدم وجود لگ هنگام تعاملات سنگین کاربری.

## ۴. معماری پروژه
این پروژه به صورت **Monorepo** (با استفاده از npm workspaces) ساختاریافته است تا هسته منطقی (Core) از رابط کاربری (UI) جدا باشد:

```text
project/
  apps/
    desktop/      # اپلیکیشن اصلی Electron (شامل Main و Renderer)
  packages/
    core/         # پکیج npm شامل تایپ‌ها، اینترفیس‌ها و منطق پایه
```

- **`packages/core`:** شامل مدل‌های داده، تایپ‌های TypeScript، و اینترفیس‌های مشترک بین فرآیندهای Main و Renderer. این پکیج قابلیت انتشار به عنوان یک ماژول مستقل npm را دارد.
- **`apps/desktop`:** شامل دو بخش `main` (برای مدیریت سیستم‌عامل، فایل‌ها و دیتابیس محلی) و `renderer` (رابط کاربری ساخته شده با React و Vite) است.

---

## ۵. تکنولوژی‌های استفاده شده
- **Electron.js:** ساخت اپلیکیشن دسکتاپ کراس‌پلتفرم.
- **React.js:** توسعه رابط کاربری (UI).
- **TypeScript:** تایپ‌سیستم قوی برای کاهش خطاها و توسعه پایدارتر.
- **Vite:** ابزار بیلد فوق‌سریع برای بخش Renderer.
- **Zustand:** مدیریت State در بخش کلاینت.
- **electron-builder:** پکیج کردن و ساخت فایل‌های نصبی (Installers).
- **Node.js:** اجرای منطق سمت بک‌اند دسکتاپ و مدیریت فایل‌سیستم.

## ۶. نحوه اجرای پروژه (Development)

برای اجرای پروژه در محیط توسعه، ابتدا وابستگی‌ها را نصب کرده و سپس دستور اجرای دولوپمنت را بزنید:

```bash
# نصب تمامی وابستگی‌های Monorepo
npm install

# اجرای پروژه در حالت توسعه
npm run dev
```
این دستور به صورت همزمان `packages/core` را بیلد کرده، Vite dev server را اجرا می‌کند و Electron را بالا می‌آورد.

---

## ۷. نحوه Build گرفتن (Production)

پروژه به گونه‌ای تنظیم شده که می‌توان خروجی‌های مختلفی برای سیستم‌عامل‌های مختلف گرفت:

```bash
# بیلد کردن پکیج Core و اپلیکیشن Desktop به صورت کامل
npm run build
```

برای ساخت فایل‌های نصبی نهایی (Installers):

```bash
# ساخت نسخه مک (Intel و Apple Silicon) و نسخه ویندوز (NSIS)
npx electron-builder --mac --win
```
خروجی‌ها در پوشه `apps/desktop/dist/` (یا پوشه `dist` روت بسته به تنظیمات) قرار می‌گیرند.

## ۸. سیستم Release

برای مدیریت نسخه‌ها و انتشار، یک اسکریپت اختصاصی توسعه داده شده است که فرآیند به‌روزرسانی ورژن‌ها در تمام پکیج‌های Monorepo، بیلد گرفتن، ساخت فایل‌های نصبی دسکتاپ و انتشار پکیج npm را خودکار می‌کند:

```bash
# برای آپدیت نسخه Patch (مثلا 1.0.0 به 1.0.1)
npm run release:patch

# برای آپدیت نسخه Minor (مثلا 1.0.0 به 1.1.0)
npm run release:minor

# برای آپدیت نسخه Major (مثلا 1.0.0 به 2.0.0)
npm run release:major
```

## ۹. نحوه استفاده از Environment ها
محیط‌ها (Environments) به شما اجازه می‌دهند متغیرهایی تعریف کنید که در سرتاسر درخواست‌ها قابل استفاده باشند.
- **تعریف متغیر:** با فرمت `{{variable_name}}` در URL، Headerها، و Body.
- **مثال:** `https://api.example.com/users/{{user_id}}`
- **قابلیت‌ها:** 
  - جابجایی سریع بین محیط‌های مختلف (مانند Staging و Production).
  - پیش‌نمایش مقادیر با قرار دادن موس روی متغیرها (Hover Preview).

---

## ۱۰. موتور اسکریپت (Script Engine)
پلتفرم از یک موتور اجرای جاوااسکریپت ایزوله استفاده می‌کند که به شما اجازه می‌دهد قبل و بعد از هر درخواست، منطق دلخواه خود را اجرا کنید.

**نمونه Pre-request Script:**
```javascript
// تنظیم یک توکن داینامیک قبل از ارسال درخواست
const timestamp = Date.now();
pm.environment.set("current_time", timestamp);
```

**نمونه Test Script:**
```javascript
// بررسی وضعیت پاسخ و مقادیر برگشتی
pm.test("Status code is 200", () => {
    pm.response.to.have.status(200);
});

pm.test("Response contains token", () => {
    const data = pm.response.json();
    pm.environment.set("auth_token", data.token);
});
```

## ۱۱. اجراکننده کالکشن (Runner)
ابزار Runner برای اجرای دسته‌ای درخواست‌ها (Batch Execution) طراحی شده است.
- **اجرای متوالی:** تمام درخواست‌های یک پوشه یا کالکشن را به ترتیب اجرا می‌کند.
- **ارزیابی خودکار:** در طول اجرا، تمام اسکریپت‌های تست (Test Scripts) اجرا شده و نتایج موفقیت/شکست ثبت می‌شود.
- **گزارش‌گیری:** در پایان اجرا، یک نمای کلی از زمان اجرا، تعداد پاس شده‌ها و خطاها نمایش داده می‌شود.

---

## ۱۲. ساختار توسعه پروژه
پروژه با معماری ماژولار و مقیاس‌پذیر طراحی شده است:
- **Event-Driven:** استفاده از یک سیستم Event Bus سبک برای ارتباط بین کامپوننت‌های UI بدون درگیری Props Drilling.
- **IPC Communication:** استفاده از ساختار Context Bridge در Preload اسکریپت برای ارتباط کاملا امن بین رندر (React) و پردازش اصلی (Node.js).
- **Clean UI:** تمامی استایل‌ها در فایل `index.css` مدیریت شده و از Utility Class ها در کنار مفاهیم پایه CSS استفاده شده تا وابستگی به فریم‌ورک‌های سنگین کاهش یابد.

## ۱۳. Roadmap و برنامه‌های آینده
مسیر توسعه این پلتفرم در آینده شامل موارد زیر است:
- [ ] **سرور Mock (Mock Server):** امکان شبیه‌سازی APIها به صورت محلی.
- [ ] **همگام‌سازی ابری (Cloud Sync):** گزینه اختیاری برای سینک کردن کالکشن‌ها بین دستگاه‌های مختلف.
- [ ] **سیستم افزونه (Plugin System):** ساختار کاملا ماژولار برای نصب افزونه‌های توسعه‌دهندگان شخص ثالث.
- [ ] **یکپارچگی CI/CD:** ارائه ابزار CLI برای اجرای خودکار Collection Runner در پایپ‌لاین‌های CI.
- [ ] **همکاری تیمی (Team Collaboration):** امکان به اشتراک‌گذاری ورک‌اسپیس‌ها در شبکه داخلی.
- [ ] **Auto Update:** پیاده‌سازی سیستم آپدیت خودکار برای نسخه‌های جدید دسکتاپ.
